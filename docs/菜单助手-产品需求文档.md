# 🥘 菜单助手 - 产品需求文档

> 一款帮助餐饮店主每日开菜单、管理库存、对接供应商的微信小程序

---

## 一、背景与目标

店主经营一家主营凉皮、肉夹馍、石锅饭的小店，每天需要手工开菜单备货（约30-50种原材料），目前纸质记录，容易漏开导致第二天缺货。

**目标：** 用微信小程序替代纸质菜单，实现智能备货建议、自动生成分类采购单、一键发给供应商。

---

## 二、用户

- 单人使用（店主本人）

## 三、功能模块

### 3.1 菜品与原材料管理

- 配置菜品（凉皮 / 肉夹馍 / 石锅饭）
- 每个菜品关联所需原材料及用量
  - 例：肉夹馍 = 馍（固定1个）+ 腊汁肉（非固定）+ 辣椒
- 原材料支持标记所属**菜品分类**：
  - 【凉皮类】【肉夹馍类】【石锅饭类】【通用食材】
- 原材料标记**消耗方式**（独立于菜品分类，两套属性并存）：
  - 即时消耗：白饼、鸡蛋等，当日采购全额计入成本
  - 摊销消耗：料子、香料等，按历史进货周期每日均摊成本
- 每种原材料可绑定供应商（或标记为"自购"）
- 设置原材料的：
  - 安全库存线（低于此值提醒）
  - 默认备货周期（如：凉皮面粉默认每2天进一次）
  - 最长存放天数（超时提醒新鲜度）
- **价格快照机制**：
  - 每种原材料预设默认单价，仅作为采购时的默认填入值
  - 修改预设单价只影响未来采购，历史账单价格永久锁定不变

---

### 3.2 原材料三种管理模式

根据食材特性，分为三种补货推算模式：

#### 模式一：固定消耗（如白饼、鸡蛋）
- 消耗关系精确：1份肉夹馍 = 1个白饼
- **前期**（无订单数据）：人工盘点库存
- **后期**（接入收钱吧）：销量 × 固定用量，精确推算库存消耗

#### 模式二：非固定消耗（如腊汁肉、凉皮面粉）
- 每份用量不固定（手工分量），无法精确计算
- **前期**：人工盘点 + 按进货时间间隔学习
  ```
  第1次进货：买25斤腊汁肉 → 记录时间戳
  第2次进货：买25斤腊汁肉 → 记录间隔天数
  → 接近历史平均间隔时，推荐面板提示补货
  ```
- **后期**（接入收钱吧）：自动切换为销量推算
  ```
  进货25斤 → 卖了182份 → 重新进货
  推算：每份消耗 ≈ 25÷182 = 0.137斤
  当累计销量接近182份时，提醒补货
  ```
- 切换后保留时间间隔历史数据作辅助参考

#### 模式三：调料类（如盐、油、料子）
- 用量难以精确追踪，按时间周期提醒
- **自学习提醒周期**：
  - 冷启动：默认5天提醒一次
  - 逐步学习：根据历史实际进货间隔，用加权移动平均修正周期
    ```
    新周期 = 最近一次×50% + 上上次×30% + 再上次×20%
    ```
- **提醒交互**（在开菜单推荐面板中展示）：
  ```
  ☑ 盐   已5天未进货
        [✅ 加入采购]  [⏰ X天后再提醒]  [❌ 忽略]
  ```
  - 加入采购 → 进货后重置计时器，记录本次实际周期
  - 填写延迟天数 → N天后再次出现
  - 忽略（不填）→ 不再主动提醒，等实际进货时自动回算实际间隔并更新均值

> **两套分类体系说明：**
> - 菜品分类（凉皮类/肉夹馍类/石锅饭类/通用食材）→ 用于采购统计
> - 消耗方式（即时/摊销）→ 用于利润计算
> - 一种食材同时拥有两个标签，互不影响

### 3.3 每日库存盘点

- 每天营业结束后快速录入剩余库存数量
- 支持快速数字键盘输入
- **定时提醒**：每天固定时间推送提醒"该盘点库存了"
- **新鲜度预警**：
  - 若某原材料已存放超过设定天数，提示"凉皮已放X天，是否还能使用？建议重新备货"
  - 即使库存仍有剩余，也会发出提醒

> **库存自动化演进路径：**
> - 前期：统一人工盘点（固定消耗与非固定消耗无区别）
> - 后期接入收钱吧：固定消耗食材自动推算，非固定消耗食材历史学习，两者自动区分

---

### 3.4 智能开菜单（核心功能）

#### 天气获取
- 自动定位获取当前位置
- 拉取未来3天天气预报
- 权重：明天（主要）> 后天（次要）> 第三天（参考）

#### AI智能推荐面板
开菜单时展示推荐列表，综合以下因素给出建议：

| 因素 | 说明 |
|------|------|
| 库存剩余 | 当前库存低于安全线 |
| 备货周期 | 超过历史平均周期未进货 |
| 天气 | 雨天↓ 晴天↑（连续雨天大幅缩减） |
| 星期/节假日 | 周末/节假日适当多备 |
| 季节 | 夏天凉皮需求旺 |
| 销量进度 | 累计销量接近历史补货点（接入收钱吧后） |

**推荐面板示例：**
```
🌦️ 天气参考：明天小雨 / 后天中雨 / 第三天晴

📌 今日建议补货

☑ 凉皮面粉   建议 1包 ↓   （原本2包，明日雨天客流少）
☑ 腊汁肉     建议 25斤    （已卖176份，接近历史补货点182份）
☑ 小葱       建议 3斤 ↓   （原本5斤，连续两天有雨）
☑ 盐         已5天未进货  [✅加入采购] [⏰X天后再提醒] [❌忽略]
☐ 大米       建议 30斤    （正常周期）

[全选推荐]  [生成菜单]
```

- 每条推荐显示理由
- 店主可勾选/取消/修改数量
- AI根据店主历史选择习惯，逐渐修正推荐准确性

### 3.5 生成分类采购菜单

确认推荐列表后，自动生成按分类整理的采购清单：

```
📋 今日采购菜单（3月31日）

【凉皮类】
- 凉皮面粉   1包
- 黄瓜       8斤

【肉夹馍类】
- 馍         80个
- 腊汁肉     25斤

【石锅饭类】
- 大米       30斤
- 鸡蛋       40个

【通用食材】
- 小葱       3斤
- 食用油     1桶
- 盐         1袋
```

- 多个菜品共用的原材料合并到【通用食材】分类

### 3.6 供应商管理与发单

#### 供应商配置
- 管理供应商列表（姓名 / 联系方式 / 负责品类）
- 每种原材料绑定供应商（或标记为"自购"）

#### 发单功能
- 采购菜单按供应商自动拆分
- **一键复制**对应供应商的订单文本，手动粘贴发微信
- 复制内容格式示例：
  ```
  📋 明日订单（3月31日）
  黄瓜 8斤
  小葱 3斤
  食用油 1桶
  ```

#### 供应商对账本
- 每种原材料预设单价，采购时自动填入（可临时修改当次价格）
- 每次进货自动生成账单记录（价格快照，历史永久不变）
- 供应商欠款总览：
  ```
  📋 A供应商对账

  上次结账：3月25日
  未结账单：
    3月26日  小计 180元
    3月28日  小计 205元
    3月30日  小计 190元
  ─────────────────
  累计未结：575元

  [查看明细]  [✅ 已结账，一键清零]
  ```
- 结账操作：全额结清，记录结账时间，账单归入历史
- 历史账单可随时查看核对

### 3.7 财务统计

#### 成本计入原则
- **权责发生制**：进货时计入成本，与结账时间无关
- **即时消耗**食材：当日采购金额全额计入当日成本
- **摊销消耗**食材（料子、香料等）：
  - 按历史进货周期每日均摊
  - 冷启动（无历史）：等第二次进货时回算实际天数补算
  - 实际使用天数超出预估时：进货当天修正，回算实际每日成本

#### 采购支出统计（两个维度）

**维度一：按菜品分类**
```
本月采购支出

【凉皮类】    1,280元  42%
【肉夹馍类】    960元  32%
【石锅饭类】    520元  17%
【通用食材】    280元   9%
─────────────────────
合计          3,040元
```

**维度二：按供应商**
```
本月采购支出

A供应商（凉皮）  1,200元
B供应商（肉类）    980元
C供应商（蔬菜）    560元
自购（市场）       300元
─────────────────────
合计             3,040元
```

**维度三：按单个原材料**
```
本月采购支出（原材料明细）

黄瓜      320元   进货8次
白饼      480元   进货12次
腊汁肉    760元   进货6次
凉皮面粉  420元   进货5次
...
─────────────────────
合计    3,040元
```
- 可查看每种原材料的历史进货价格走势（展示用）

- 时间维度：今日 / 本周 / 本月（三个维度均支持）

#### 营业收入
- 每日手动录入当天营业额
- 统计维度：今日 / 本周 / 本月 总收入

#### 利润分析
```
今日毛利润 = 今日营业收入
           - 今日即时消耗成本
           - 今日应摊销成本（各摊销食材当日份额之和）
```
- 日 / 周 / 月 维度展示

### 3.8 提醒系统

| 提醒类型 | 触发条件 | 时机 |
|---------|---------|------|
| 库存盘点提醒 | 每天固定时间 | Push通知 |
| 新鲜度预警 | 存放超过设定天数 | 开菜单时推荐面板 |
| 补货周期提醒 | 超过历史平均周期未进货 | 开菜单时推荐面板 |
| 调料周期提醒 | 距上次进货超过学习周期 | 开菜单时推荐面板 |
| 节假日/周末提醒 | 明日为节假日/周末 | 开菜单时推荐面板 |

---

### 3.9 第三方系统对接（后期）

#### 收钱吧扫码点餐系统
- 对接后自动获取各菜品实时销量
- 固定消耗食材：销量 × 固定用量，自动推算库存消耗
- 非固定消耗食材：切换为销量学习模式，保留历史时间数据辅助参考
- 减少人工盘点频率，库存数据准确性大幅提升

> **注意：** 即使接入收钱吧，非固定消耗食材（腊汁肉等）的库存仍为估算值，仅供参考，建议定期人工校对

## 六、已解决的逻辑冲突

| 冲突 | 解决方案 |
|------|---------|
| 成本计入时机 | 进货时计入（权责发生制），与结账时间无关 |
| 两套分类体系并存 | 菜品分类（统计用）与消耗方式（利润用）独立存在，互不影响 |
| 摊销超期处理 | 进货当天回算实际天数，修正之前的摊销数据 |
| 前期固定/非固定食材区分 | 前期统一人工盘点，接入收钱吧后自动区分 |
| 两阶段学习切换 | 接入收钱吧后自动切换，保留时间数据作辅助参考 |
