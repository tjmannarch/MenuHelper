# 菜单助手 - 需求分析文档

> 基于《菜单助手-产品需求文档》结构化拆解，供后续 CleanDDD 建模使用。

---

## 一、干系人表

| 角色 | 目标/痛点 | 权限/限制 | 备注 |
|------|---------|---------|------|
| 店主 | 每天手工开菜单耗时、容易漏开导致缺货；需要智能备货建议和采购单 | 系统唯一用户，拥有全部操作权限 | 单人使用，无多角色场景 |
| 供应商（被动） | 接收采购订单 | 无系统账号，通过微信被动接收订单文本 | 不直接使用系统 |
| 收钱吧系统（后期） | 提供销量数据 | 只读数据源，通过 API 对接 | P3 后期功能 |

## 三、业务实体视图

| 业务实体 | 覆盖需求条目 | 主要职责/规则 | 关键输入/输出 |
|---------|------------|------------|------------|
| 菜品 | REQ-001、REQ-002 | 维护菜品与原材料的关联关系及用量 | 菜品名、原材料列表（含用量类型） |
| 原材料 | REQ-003～REQ-009 | 维护原材料的分类、消耗方式、库存参数、供应商绑定、预设单价 | 分类标签、消耗方式、安全库存线、备货周期、存放天数、默认单价 |
| 库存盘点 | REQ-010～REQ-012 | 记录每天各原材料的实际剩余库存；新鲜度预警 | 盘点日期、原材料、剩余数量 |
| 开菜单 | REQ-013～REQ-018 | 综合多因素生成补货推荐列表；记录店主选择历史 | 天气数据、库存、周期、日期 → 推荐列表 |
| 采购单 | REQ-019～REQ-020、REQ-022 | 由确认后的推荐列表生成；按分类和供应商拆分 | 推荐列表 → 分类采购单、供应商订单文本 |
| 供应商 | REQ-021、REQ-025～REQ-027 | 维护供应商信息；追踪未结账单和欠款 | 供应商信息、账单列表 |
| 账单 | REQ-023～REQ-024、REQ-027 | 每次进货生成价格快照账单；历史永久锁定 | 进货日期、原材料、数量、实际单价（快照） |
| 营业收入 | REQ-031～REQ-032 | 记录每日手动录入的营业额 | 日期、营业额 |
| 财务统计 | REQ-028～REQ-033 | 按多维度汇总采购支出和利润 | 账单、营业收入、摊销成本 → 统计数据 |
| 成本摊销 | REQ-034 | 摊销消耗食材按周期每日均摊；冷启动回算 | 进货记录、历史进货间隔 → 每日摊销额 |
| 补货学习 | REQ-035～REQ-038 | 学习各食材的进货间隔/销量，持续修正推荐 | 进货记录、销量数据 → 推荐权重 |
| 提醒 | REQ-011～REQ-012、REQ-017 | 定时推送盘点提醒；新鲜度/周期提醒在推荐面板展示 | 时间配置、库存、进货记录 → 提醒触发 |
| 天气数据 | REQ-013 | 调用第三方 API 获取未来3天天气；影响推荐权重 | 位置 → 天气预报 |

## 五、业务规则与约束

| 规则/约束 | 相关实体 | 依赖系统/数据 | 备注 |
|---------|---------|------------|------|
| 历史账单价格永久锁定，修改预设价不影响历史 | 账单、原材料 | — | 价格快照机制 |
| 成本计入采用权责发生制（进货时计入） | 账单、财务统计 | — | — |
| 即时消耗：当日进货金额全额计入当日成本 | 成本摊销 | — | — |
| 摊销消耗：按历史周期每日均摊；冷启动时等第二次进货后回算 | 成本摊销 | — | 摊销超期时进货当天修正并回算 |
| 两套分类体系（菜品分类 & 消耗方式）独立，互不影响 | 原材料 | — | — |
| 调料学习周期公式：最近×50% + 上上次×30% + 再上次×20% | 补货学习 | — | 冷启动默认5天 |
